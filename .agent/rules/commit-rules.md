To ensure seamless collaboration with high-level AI agents like Claude Opus and to maintain a professional development history for Wantoog, here are the English-only commit rules.You can copy and paste this into your project-docs.md or a separate CONTRIBUTING.md file.Commit Message Convention1. StructureThe commit message should follow this format:Plaintext<type>: <description>
2. Commit TypesTypeDescriptionfeatA new feature for the userfixA bug fix for the userdocsChanges to the documentation (project-docs.md, etc.)styleUI/UX changes, formatting, missing semi-colons (no logic change)refactorRefactoring code that neither fixes a bug nor adds a featureassetAdding or updating 3D models, textures, or sound fileschoreUpdating build tasks, package manager configs, etc.testAdding missing tests or correcting existing tests3. Project-Specific Context TagsFor Wantoog, use these bracketed tags at the beginning of the description to help AI agents understand the functional domain:[AI]: Classification models, taste profiling, and insights.[3D]: Diorama rendering, SceneKit/RealityKit, and shaders.[DATA]: Database schema, local storage, and data normalization.4. Writing RulesLanguage: Always use English for commit messages.Mood: Use the imperative, present tense (e.g., "Add feature" not "Added feature").Capitalization: Do not capitalize the first letter of the description.Punctuation: Do not end the subject line with a period.Atomicity: One commit should represent one logical change.Examplesfeat: [3D] add steam particle effect to ttukbaegi modelstyle: [UI] apply warm-toned background texture to main viewfix: [AI] correct misclassification between tonkotsu and shio ramendocs: update project roadmap in project-docs.mdasset: add 3D isometric chair for ramen shop dioramarefactor: [DATA] decouple taste analysis logic from view model
